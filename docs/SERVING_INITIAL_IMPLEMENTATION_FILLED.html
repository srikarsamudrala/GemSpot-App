3) Interpretation And Recommendation

  • Best throughput: baseline_http
  • Best latency: onnx_cpu
  • Best overall recommended deployment: onnx_cpu

Reasoning: baseline_http provides the highest throughput and simplest deployment. onnx_cpu cuts latency substantially versus both XGBoost variants. onnx_workers4 only slightly improves throughput over onnx_cpu, so onnx_cpu is the clearest recommended option on the current VM flavor. The infra_rightsized run shows that moving the ONNX configuration to the second VM flavor did not materially improve latency or throughput, which supports keeping the smaller CPU flavor as the better right-sized choice.

4) Right-Sizing Notes

  • Primary VM flavor: baseline Chameleon CPU flavor used for the first set of measurements
  • Comparison VM flavor: assumed m1.large based on the second lease context
  • CPU usage under representative load: collected during docker stats and top
  • RAM usage under representative load: collected during free -h
  • GPU usage: N/A
  • Final recommendation: the ONNX configuration on the smaller CPU flavor is the better right-sized deployment because the larger VM did not produce a meaningful latency or throughput gain
